# Project Instructions

## Repository Rules

### Project Structure

- Shared configuration distribution source — no application code.
- Deliverables: APM packages, reusable GitHub Actions workflows, Renovate policy presets.

### Package Editing

- Source of truth: `.apm/packages/<name>/`. Edit there.
- Run `apm install --update` after edits to propagate to `.github/`, `.agents/`, `.kiro/`.
- Never edit `.agents/`, `.kiro/`, `apm_modules/` directly — generated by APM.

### Workflow Conventions

- Reusable workflows use `workflow_call` trigger.
- File names: `ci-*` (CI), `cd-*` (CD), `on-*` (event-triggered callers).
- Keys in `inputs`, `env`, `permissions`, `with` must be alphabetically ordered.

### Build and Validation

```bash
apm install --update
apm audit --ci
actionlint
ghalint run
zizmor .github/workflows/
```

### Temporary Artifacts

Write to `tmp/`.

### Documentation Editing

When structure or package composition changes, keep `README.md` and `docs/` up to date.

## Security Guidelines

- No secrets in source code, logs, or test data.
- No destructive operations as defaults in command examples.
