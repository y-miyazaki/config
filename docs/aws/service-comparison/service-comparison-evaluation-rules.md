## 評価の原則

- [共通評価ルール](../../tool-comparison/tool-comparison-evaluation-rules.md) の原則を継承する
- AWS サービス間の比較であるため、提供元は全て AWS となる。代わりに「サービスカテゴリ」「課金モデル」を重視する
- ワークロード特性 (トラフィックパターン、データ量、レイテンシ要件) に基づいて選定する
- マネージド度合いと運用負荷のトレードオフを明示する
- 将来のスケール要件を考慮し、移行コストも評価する

## 記載ルール

| ルール                   | 記載方法                                                                      |
| ------------------------ | ----------------------------------------------------------------------------- |
| 比較表の基本形           | `\| 比較項目 \| A \| B \| ... \|` の Markdown table を使用する                |
| 先頭行の順序             | 1行目 `サービスカテゴリ`、2行目 `ドキュメント`、3行目 `課金モデル`            |
| 4行目以降の推奨順序      | `マネージド度` → `主用途` → 追加評価項目                                      |
| 二値の表現               | ✅ / ⚠️ / ❌ を使い、意味を混在させない                                        |
| 記号の意味               | ✅ = 満たす / ⚠️ = 条件付き・限定的 / ❌ = 満たさない                           |
| 定量値の表現             | 実数・件数・料金・URL はそのまま記載し、記号で置き換えない                    |
| 定性的な比較             | `高 / 中 / 低` や短い名詞句で統一し、長文説明は Guidelines に寄せる           |
| 結論の書き方             | `**→` で始める1行の要約を置き、その後に箇条書きを続ける                        |
| 追加説明の書き方         | 比較表に入らない前提や除外項目は導入文か `## 補足` で明示する                  |

## 必須評価軸 (MUST)

[共通評価ルール](../../tool-comparison/tool-comparison-evaluation-rules.md) の必須評価軸に加え、以下を AWS サービス比較固有の必須評価軸とする。

| 評価軸              | 観点                                                                 | 判定例                            |
| ------------------- | -------------------------------------------------------------------- | --------------------------------- |
| Workload Fit        | ワークロード特性 (常時稼働/バースト/バッチ) に適合するか             | 最適 / 適合 / 不適合              |
| Scalability         | スケール方式・上限・速度が要件を満たすか                             | 自動/手動、上限値                 |
| Availability        | SLA・マルチ AZ・リージョン冗長の要件を満たすか                       | SLA 値、冗長構成                  |
| Operational Burden  | 運用負荷 (パッチ適用、監視、障害対応) の大きさ                       | 低 / 中 / 高                      |
| Cost Model          | 課金モデルが利用パターンに適合するか                                 | 従量/固定/予約、概算月額          |
| Migration Cost      | 他サービスへの移行難易度                                             | 低 / 中 / 高                      |

## 加点評価軸 (SHOULD)

| 評価軸              | 観点                                          |
| ------------------- | --------------------------------------------- |
| IaC Support         | Terraform/CDK/CloudFormation での管理容易性   |
| Observability       | CloudWatch/X-Ray 統合、メトリクス充実度       |
| Security Posture    | VPC 統合、暗号化、IAM 粒度                    |
| Ecosystem           | 他 AWS サービスとの連携容易性                 |
| Developer Experience | ローカル開発、デバッグ、デプロイ速度          |

## 判定フロー

1. ワークロード特性 (常時稼働/バースト/バッチ/イベント駆動) を明確にする
2. 非機能要件 (SLA、レイテンシ、スループット) を定義する
3. 候補サービスごとに必須評価軸を判定し、不適合なら除外する
4. 残った候補を加点評価軸で比較する
5. コスト試算を行い、利用パターン別に比較する
6. 不採用サービスの理由を1行で記録する

## スコアリング方式

[共通評価ルール](../../tool-comparison/tool-comparison-evaluation-rules.md) のスコアリング方式を継承する。

| スコア | 意味               |
| ------ | ------------------ |
| 3      | 要件を十分に満たす |
| 2      | 条件付きで満たす   |
| 1      | 一部のみ満たす     |
| 0      | 満たさない         |

重み付けの推奨:

- 必須評価軸: 合計 70%
- 加点評価軸: 合計 30%

## 記載テンプレート

各 comparison ドキュメントの Guidelines には以下を含める。

- 選定方針: 「何を優先して選ぶか」
- 採用条件: 「どの条件なら採用するか」
- 不採用条件: 「どの条件なら採用しないか」
- 併用条件: 「単体でなく併用する条件」
- 更新条件: 「いつ再評価するか」
