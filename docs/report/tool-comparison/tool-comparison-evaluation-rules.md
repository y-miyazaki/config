## 評価の原則

- 単一ツールの押し付けを避け、要件適合で選ぶ
- 代替関係と補完関係を明示する
- 採用理由だけでなく「不採用理由」も記載する
- 結論は「条件付き推奨」で表現する
- 価格・機能は変動するため、更新日を残す

## 記載ルール

| ルール               | 記載方法                                                                                                                                                                                                                                                                                                     |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 比較表の基本形       | `\| 比較項目 \| A \| B \| ... \|` の Markdown table を使用する                                                                                                                                                                                                                                               |
| 比較対象の並び順     | ヘッダ行の比較対象 (ツール名) は A–Z 順に並べる                                                                                                                                                                                                                                                              |
| 先頭行の順序         | 比較対象に存在する場合、1 行目を `提供元`、2 行目を `リポジトリ`、3 行目を `ドキュメント`、4 行目を `ライセンス` にする。存在しない項目や公開 repo がない項目は無理に追加せず `-` にする。AI Agent 等リポジトリ非公開でも独自の仕様サイトやドキュメントサイトがある場合は `ドキュメント` 行に URL を記載する |
| 5 行目以降の推奨順序 | `提供形態` → `主用途` → `実行例` → 追加評価項目（存在する場合）                                                                                                                                                                                                                                              |
| 二値の表現           | ✅ / ⚠️ / ❌ を使い、意味を混在させない                                                                                                                                                                                                                                                                      |
| 記号の意味           | ✅ = 満たす / ⚠️ = 条件付き・限定的 / ❌ = 満たさない                                                                                                                                                                                                                                                        |
| 定量値の表現         | 実数・件数・料金・URL はそのまま記載し、記号で置き換えない                                                                                                                                                                                                                                                   |
| 定性的な比較         | `高 / 中 / 低` や短い名詞句で統一し、長文説明は Guidelines に寄せる                                                                                                                                                                                                                                          |
| 結論の書き方         | `**→` で始める 1 行の要約を置き、その後に箇条書きを続ける                                                                                                                                                                                                                                                    |
| 追加説明の書き方     | 比較表に入らない前提や除外項目は導入文か `## 補足` で明示する                                                                                                                                                                                                                                                |
| 参照リンクの書き方   | 比較対象の別ドキュメントは相対リンクで参照する                                                                                                                                                                                                                                                               |

## 必須評価軸 (MUST)

| 評価軸          | 観点                                                            | 判定例                               |
| --------------- | --------------------------------------------------------------- | ------------------------------------ |
| Problem Fit     | 課題に直接効くか。責務がずれていないか                          | 直接解決 / 部分解決 / 非対応         |
| Security        | 脆弱性検知・誤検知制御・監査証跡・権限制御の実装可否            | 要件を満たす / 追加対策が必要 / 不足 |
| Operation Model | SaaS/セルフホスト要件、ネットワーク制約、データ所在要件への適合 | 適合 / 条件付き適合 / 不適合         |
| Integration     | 既存 CI/CD・SDK・言語・クラウドとの統合難易度                   | 低 / 中 / 高                         |
| Cost/TCO        | ライセンス費、運用工数、誤検知対応コスト、学習コスト            | 低 / 中 / 高                         |
| Maintainability | バージョン追従、設定の見通し、ロック再現性、保守体制            | 高 / 中 / 低                         |

## 加点評価軸 (SHOULD)

| 評価軸               | 観点                                         |
| -------------------- | -------------------------------------------- |
| Developer Experience | 修正導線、IDE 連携、フィードバック速度       |
| Ecosystem Maturity   | コミュニティ規模、更新頻度、ドキュメント品質 |
| Vendor Lock-in       | 乗り換えコスト、データ可搬性、API 互換性     |
| Extensibility        | カスタムルール、プラグイン、API 拡張性       |

## 判定フロー

1. 要件を必須要件/任意要件に分離する。
2. 候補ツールごとに必須評価軸を判定し、1 つでも不適合なら除外する。
3. 残った候補を加点評価軸で比較する。
4. 代替/補完を判定し、単体採用か併用かを決める。
5. 導入しないツールの理由を 1 行で記録する。

## スコアリング方式

| スコア | 意味               |
| ------ | ------------------ |
| 3      | 要件を十分に満たす |
| 2      | 条件付きで満たす   |
| 1      | 一部のみ満たす     |
| 0      | 満たさない         |

重み付けの推奨:

- 必須評価軸: 合計 70%
- 加点評価軸: 合計 30%

判定閾値の推奨:

- 採用候補: 加重平均 2.3 以上
- 条件付き採用: 1.8 以上 2.3 未満
- 不採用: 1.8 未満

## 記載テンプレート

各 comparison ドキュメントの Guidelines には以下を含める。

- 選定方針: 「何を優先して選ぶか」
- 採用条件: 「どの条件なら採用するか」
- 不採用条件: 「どの条件なら採用しないか」
- 併用条件: 「単体でなく併用する条件」
- 更新条件: 「いつ再評価するか」
